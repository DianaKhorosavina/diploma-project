# Автоматизируемые сценарии

## Входные данные 

Разработчиками были заложены следующие условия валидных данных для проведения тестирования: 
1. Номера карт: 4444 4444 4444 4441 (валидный номер карты по которому банк одобряет заявку на покупку тура); 4444 4444 4444 4442 (невалидный номер карты, по которому банк отказывает в покупке тура).
2. Поле "Месяц": валидное значение для данного поля - текущий месяц проведение тестов
3. Поле "Год": валидное значение для данного поля - текущий год проведения тестов
4. Поле "Владелец": валидное значение для данного поля - имя и фамилия владельца карта на латинице
5. Поле "CVC/CVV": валидное значение для данного поля - любые три цифры.

# Перечень используемых инструментов

1. Selenide - для удобства работы с PageObject 
2. Lombok - для уменьшения количества кода, который необходимо написать для работы автотестов
3. JavaFaker - для генерации информации о пользователе
4. MySQL - для работы с базами данных
5. PostgreSQL - для работы с базами данных
6. Docker - для подготовки и использования контейнеров эмулирующих окружение, что позволяет работать с базами данных и банковским сервисом 
7. Allure - для подробных и удобных для чтения и понимания отчетов по итогам тестов.


# Возможные риски

При создании автотестов можно столкнуться со следующими проблемами:
1. Отсутствие доступа к базам данных
2. Отсутствие разрешения на проведение автотестов
3. Последующие изменения в веб-приложении которые могут привести к падению уже успешно пройденных тестов
4. Отсутствие должного финансирования для создания тестов
5. Отсутствие в команде специалиста умеющего работать с автотестами
6. Автоматизация сценариев которые проще и выгоднее выполнять вручную
